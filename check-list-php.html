Application en SAAS

  • Modèle commercial dans lequel les logiciels sont installés sur des serveurs distants
  • L'un des avantages est une meilleur maîtrise des coûts par utilisateur (licence, maintenance, infrastructure)
  • Possibilité de déployer très rapidement une solution qui n'a besoin d'aucune adaptation
  • Par contre, les données de l'entreprise cliente de la solutions SAAS sont généralement stockées chez le prestataire

Exemple de frameworks

  • Framework open source
  • Avec une grande communauté
  • Documentation importante (cookbook, etc...)
  • Nécessite un investissement important pour le prendre en main
  • Assez lourd donc à éviter pour les projets à court terme
  • Framework open source
  • Avec une grande communauté
  • Documentation moins présente que sur les autres frameworks
  • Nécessite un investissement moindre pour débuter
  • Assez lourd donc à éviter pour les projets à court terme

Docker

  • L'application se trouve dans un container
  • Le container est plus léger que la machine virtuelle
  • Stack applicative plus proche de celle qui est en production
  • Les containers crées sous Linux ne pourront pas être portés sur Windows Server et inversement
